# Worker Verification Flow — Build Guide
### GhostDetect · Mobile PWA · For Antigravity

---

## Overview

This is a SEPARATE surface from the admin dashboard.
Workers never touch the admin dashboard.
They receive a unique link via SMS/WhatsApp and complete verification on their phone.

**URL structure:** `https://ghostdetect.app/verify?token=UNIQUE_TOKEN`

Each token is:
- Single-use (burns after completion or expiry)
- Tied to one worker record in MongoDB
- Expires after 48 hours
- Generated by admin when they initiate a verification batch

---

## The 6 Screens (Build in This Order)

```
Screen 1 — Landing        (branded splash with ministry name)
Screen 2 — Staff ID       (enter ID + date of birth)
Screen 3 — OTP            (6-digit code sent to registered phone)
Screen 4 — Selfie         (camera access, face positioning)
Screen 5 — Liveness       (4 random prompts — smile, blink, turn left, turn right)
Screen 6 — Result         (VERIFIED ✅ or FAILED ❌)
```

## Tech Stack for This Flow

| Layer | Tech | Why |
|-------|------|-----|
| Framework | React + Vite (PWA) | Works in mobile browser, no app install |
| Camera | `getUserMedia` API | Built into every modern browser |
| Face detection | face-api.js (CDN) | Lightweight, runs in browser, no server needed |
| Liveness | Random prompt + expression detection | Simple, effective, impressive |
| OTP | Backend generates + sends via Termii or Africa's Talking | Nigerian SMS delivery |
| Token validation | JWT with 48h expiry | Secure single-use links |
| State | React useState (no Redux) | Keep it simple |

### Screen 4 — Selfie

**What it does:**
1. Request camera permission (`getUserMedia({ video: { facingMode: 'user' } }}`)
2. Show live video feed inside an oval frame
3. Animated scanning bar over the face
4. Corner bracket overlays
5. Tips: good lighting, face forward, no glasses
6. "Take Selfie" button → captures a frame → sends to backend for face detection

**Camera component:**
```javascript
// hooks/useCamera.js
export function useCamera() {
  const videoRef = useRef(null);
  const [stream, setStream] = useState(null);
  const [error, setError] = useState(null);

  const start = async () => {
    try {
      const s = await navigator.mediaDevices.getUserMedia({
        video: { facingMode: 'user', width: 640, height: 480 }
      });
      videoRef.current.srcObject = s;
      setStream(s);
    } catch (err) {
      setError('Camera access denied. Please allow camera and refresh.');
    }
  };

  const capture = () => {
    const canvas = document.createElement('canvas');
    canvas.width = videoRef.current.videoWidth;
    canvas.height = videoRef.current.videoHeight;
    canvas.getContext('2d').drawImage(videoRef.current, 0, 0);
    return canvas.toDataURL('image/jpeg', 0.8);
  };

  const stop = () => {
    stream?.getTracks().forEach(t => t.stop());
  };

  return { videoRef, start, capture, stop, error };
}
```

**CRITICAL:** Always call `stop()` when leaving this screen or the camera stays on.

### Screen 5 — Liveness Check (Most Important Screen)

**The 4 random prompts — pick 4 randomly from this pool:**

```javascript
const PROMPTS = [
  { id: 'smile',      emoji: '😄', text: 'Please SMILE now',           sub: 'Hold your smile for 2 seconds' },
  { id: 'blink',      emoji: '😑', text: 'Please BLINK twice',         sub: 'Blink both eyes clearly' },
  { id: 'turn_left',  emoji: '👈', text: 'Turn your head LEFT',        sub: 'Slowly turn to your left' },
  { id: 'turn_right', emoji: '👉', text: 'Turn your head RIGHT',       sub: 'Slowly turn to your right' },
  { id: 'open_mouth', emoji: '😮', text: 'Please OPEN your mouth',     sub: 'Open wide for 1 second' },
  { id: 'eyebrows',   emoji: '🤨', text: 'RAISE your eyebrows',        sub: 'Raise both eyebrows up' },
  { id: 'nod',        emoji: '🙆', text: 'NOD your head up and down',  sub: 'Two slow nods' },
];

// On mount — pick 4 randomly
const selected = PROMPTS.sort(() => Math.random() - 0.5).slice(0, 4);
```

**Why this defeats fraud:**
The prompts are randomly selected at runtime. An attacker with a printed photo or pre-recorded video cannot predict which 4 prompts will appear, making spoofing essentially impossible without physically present.

**Timer per prompt:** 4 seconds. If not completed → mark as failed.

**Implementation (simplified for demo):**
```javascript
// For the hackathon demo — use a timer-based pass
// In production you'd use face-api.js expression detection

const [currentPrompt, setCurrentPrompt] = useState(0);
const [results, setResults] = useState([]);

useEffect(() => {
  const timer = setTimeout(() => {
    // Demo: auto-pass after 4 seconds
    setResults(prev => [...prev, { prompt: prompts[currentPrompt].id, passed: true }]);
    if (currentPrompt < 3) setCurrentPrompt(prev => prev + 1);
    else navigate to result screen;
  }, 4000);
  return () => clearTimeout(timer);
}, [currentPrompt]);
```

**For real face-api.js integration (add if time allows):**
```javascript
import * as faceapi from 'face-api.js';

// Load models on mount
await faceapi.nets.tinyFaceDetector.loadFromUri('/models');
await faceapi.nets.faceExpressionNet.loadFromUri('/models');

// Detect expressions every 100ms
const detection = await faceapi
  .detectSingleFace(videoRef.current, new faceapi.TinyFaceDetectorOptions())
  .withFaceExpressions();

// Check if expression matches prompt
if (prompt.id === 'smile' && detection.expressions.happy > 0.7) → pass
if (prompt.id === 'open_mouth' && detection.expressions.surprised > 0.6) → pass
```

**Progress indicator:** 4 small bars at top — fill red as each prompt completes.

## OTP SMS — Use Termii (Nigerian provider)

```javascript
// backend/services/smsService.js
async function sendOTP(phone, otp) {
  const res = await fetch('https://api.ng.termii.com/api/sms/send', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      to: phone,
      from: 'GhostDetect',
      sms: `Your GhostDetect verification code is: ${otp}. Valid for 5 minutes. Do not share.`,
      type: 'plain',
      api_key: process.env.TERMII_API_KEY,
      channel: 'generic'
    })
  });
  return res.json();
}
```

**Store OTP in MongoDB with expiry:**
```javascript
await OTPRecord.create({
  token: verifyToken,
  otp: hashedOTP,           // hash it — never store plain
  expiresAt: Date.now() + 5 * 60 * 1000,
  used: false
});
```

## 🔴 Critical Rules for This Flow

1. **Mobile-only layout.** Max width 430px. Everything must work with thumbs. Minimum tap target 44px height.

2. **One step per screen.** Never show two steps at once. No sidebars. No tables. No headers with navigation.

3. **Progress bar always visible** at the top. Workers need to know how far they are.

4. **Camera MUST stop** when navigating away from Selfie or Liveness screens. Leaving the camera on is a UX bug and a privacy issue.

5. **Token validation on EVERY step.** If the token has expired mid-flow, show an error immediately and stop.

6. **OTP must be hashed in MongoDB.** Never store `otp: '472831'`. Always `otp: bcrypt.hashSync('472831', 10)`.

7. **The liveness prompts MUST be random.** Never hardcode the sequence. Pick 4 from the pool randomly on Screen 5 mount. This is the security mechanism.

8. **Result screen must POST to backend before rendering.** Never show "Verified" before the backend confirms. The sequence is: complete prompts → POST result → wait for response → render result.

9. **The verification token is single-use.** After `POST /verify/complete`, mark the token as `used: true` in MongoDB. Any subsequent request with the same token returns an error.

10. **Face-api.js models go in `/public/models/`.** They are large — load them lazily only when Screen 4 mounts, not on app load.

## Common Errors to Avoid

| Error | Fix |
|-------|-----|
| Camera not working on iOS Safari | Add `playsInline` attribute to video element |
| `getUserMedia` blocked on HTTP | Always use HTTPS in production (Vercel handles this) |
| OTP not delivering in Nigeria | Use Termii, not Twilio — better Nigerian coverage |
| face-api.js models 404 | Put models in `/public/models/` not `/src/` |
| Token expired mid-flow | Check token validity at each step, not just landing |
| Same link used twice | Check `token.used === true` on every backend call |
| Camera stays on after navigating | Call `stream.getTracks().forEach(t => t.stop())` on unmount |
